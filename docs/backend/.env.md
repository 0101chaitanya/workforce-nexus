# backend/.env

## 1. Overview
The `backend/.env` file contains runtime configuration settings and secret keys used by the backend Node.js application. Keeping these configurations in a `.env` file prevents sensitive details (like database credentials and encryption keys) from being hardcoded into the source code.

## 2. Key Responsibilities & Flow
The environment variables define the following parameters:
- `MONGODB_URI`: The connection string for the MongoDB instance (e.g. MongoDB Atlas). Used by `src/config/db.js` to establish connection.
- `FRONTEND_URL`: The origin URL of the React frontend application (typically `http://localhost:5173`). Used by CORS configuration in `src/index.js` to authorize requests.
- `PORT`: The TCP port on which the Express server listens (typically `3000`). Used by `src/index.js`.
- `JWT_TOKEN`: Secret key used for signing Short-Lived Access Tokens.
- `JWT_REFRESH_TOKEN`: Secret key used for signing Long-Lived Refresh Tokens.
- `EMAIL`: The SMTP email address used for sending emails (e.g. system notifications, OTPs).
- `PASSWORD`: The SMTP email password / application password.
- `NODE_ENV`: Runtime environment configuration (`development` or `production`).

## 3. Code Patterns & Best Practices
- **Environment Isolation**: Ensures different environments (dev, staging, production) can run without modifying application code.
- **Secrets Management**: This file is excluded from Git to prevent exposing credentials.

## 4. Frontend-to-Backend / Backend-to-Frontend Mapping
- `FRONTEND_URL` maps directly to the address where the frontend client is served.
- `PORT` maps to the destination port configured in the frontend's Vite proxy or the `VITE_BACKEND_URL` environment variable inside the frontend client.

## 5. Line-by-Line Code Explanation

### Basic Function of Key Logical Parts
1. **Database & Networking Config**: Defines MongoDB URI and backend port/frontend URL.
2. **Security & Auth Tokens**: Contains JWT secrets for access and refresh tokens.
3. **Email & Environment**: Specifies mail sender configurations and node environment settings.

- **Lines 1-3 (Database & Networking Config)**:
  - **Basic Function**: Configure database connection URI and basic network communication settings.
  - **Detailed Explanation**: `MONGODB_URI` (Line 1) points to the MongoDB Atlas cluster containing database `emsV1`. `FRONTEND_URL` (Line 2) sets the allowed origin for CORS requests, and `PORT` (Line 3) specifies the port number (3000) that the backend server binds to.
- **Lines 4-5 (Security & Auth Tokens)**:
  - **Basic Function**: Configure secret keys for token generation.
  - **Detailed Explanation**: `JWT_TOKEN` (Line 4) and `JWT_REFRESH_TOKEN` (Line 5) are the secret keys used by `jsonwebtoken` in `src/utils/generateTokens.js` to sign and verify access and refresh tokens.
- **Lines 6-8 (Email & Environment)**:
  - **Basic Function**: Configure SMTP email credentials and the Node runtime environment.
  - **Detailed Explanation**: `EMAIL` (Line 6) and `PASSWORD` (Line 7) are SMTP credentials used by Nodemailer in `src/utils/sendEmail.js` to send transactional emails, and `NODE_ENV` (Line 8) defines the current environment as `development`.
