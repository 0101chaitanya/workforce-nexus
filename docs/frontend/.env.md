# frontend/.env

## 1. Overview
`frontend/.env` defines environment variables for the frontend build system (Vite), specifying configurations like the backend service entry points.

## 2. Key Responsibilities & Flow
- **`VITE_BACKEND_URL`**: Specifies the base URL of the backend API (defaults to `http://localhost:3000/api`).
- Variables prefixed with `VITE_` are automatically exposed by Vite on the `import.meta.env` object, making them available in client-side React code.

## 3. Code Patterns & Best Practices
- **Vite Variable Prefixing**: Adheres to the requirement of prefixing client-facing variables with `VITE_` to prevent accidental exposure of sensitive system environment variables to the browser bundle.
- **Development vs Production**: The `.env` file typically holds development defaults, whereas production environments inject different URLs via hosting providers.

## 4. Frontend-to-Backend / Backend-to-Frontend Mapping
- **API Mapping**:
  - `import.meta.env.VITE_BACKEND_URL` -> Used inside Axios configuration (`axiosInterceptors.js`) to set the `baseURL` for all outgoing frontend HTTP requests targeting the Express server.
