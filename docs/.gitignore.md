# .gitignore

## 1. Overview
The `.gitignore` file specifies intentionally untracked files and directories that Git should ignore. This prevents temporary build artifacts, local secret credentials, dependency caches, and editor-specific directories from being committed to source control.

## 2. Key Responsibilities & Flow
The file ignores the following main groups:
- **Dependency Directories**: Excludes `node_modules` folders in the root, `frontend/`, and `backend/` directories to prevent massive third-party package folders from committing.
- **Environment Variables / Secrets**: Excludes local environment configurations (`frontend/.env`, `backend/.env`, and any files matching `*.local`) to protect API keys, database credentials, and secret tokens.
- **Build Artifacts**: Excludes `dist` (distribution build outputs) and `dist-ssr`.
- **Logs**: Excludes `logs` and all `*.log` files (e.g. Express combined logs, debug output, etc.).
- **IDE/Editor Specifics**: Excludes JetBrains files (`.idea`), VS Code configurations (`.vscode/*` except `extensions.json`), and general system file metadata (`.DS_Store`).

## 3. Code Patterns & Best Practices
- Uses standard glob patterns (e.g. `*.log` for log files, `**/.history/` for nested VS Code local histories).
- Protects secrets (`.env`) from accidentally leaking onto Git repository servers.

## 4. Frontend-to-Backend / Backend-to-Frontend Mapping
- Applies to both `frontend/` and `backend/` directory trees by ignoring code dependencies (`node_modules`), logs, and local environment files (`.env`).

## 5. Line-by-Line Code Explanation

### Basic Function of Key Logical Parts
1. **Dependency & Secret Ignores**: Excludes dependency folders and env secret files.
2. **Editor & OS Ignores**: Ignores IDE files and system configuration files.
3. **Log File Ignores**: Prevents committing development log files.
4. **Build & Temporary Files**: Excludes compiled production build directories and temp files.

- **Lines 1-8 (Dependency & Secret Ignores)**:
  - **Basic Function**: Exclude package dependencies and sensitive environment credentials.
  - **Detailed Explanation**: Ignores `.idea` (Line 1), `node_modules` (Line 3), and specific environment files `frontend/.env` (Line 5) and `backend/.env` (Line 6) to keep local secrets private, alongside subfolder dependency trees (Lines 7-8).
- **Lines 10-14 (Editor & OS Ignores)**:
  - **Basic Function**: Omit editor-specific configuration directories and macOS metadata.
  - **Detailed Explanation**: Excludes Visual Studio Code metadata files (Line 11), local files history logs (Lines 12-13), and macOS system files `.DS_Store` (Line 14).
- **Lines 18-25 (Log File Ignores)**:
  - **Basic Function**: Ignore server transaction outputs and debug files.
  - **Detailed Explanation**: Prevents committing the `logs` folder (Line 19), general `*.log` files (Line 20), and specific NPM/Yarn/Pnpm/Lerna package manager logs (Lines 21-25) generated during local testing.
- **Lines 27-31 (Build & Temporary Files)**:
  - **Basic Function**: Omit node dependencies and compiled static directories.
  - **Detailed Explanation**: Excludes compiled build distribution folders `dist` (Line 28), Server-Side Rendering output `dist-ssr` (Line 29), and any local environment settings matching `*.local` (Line 30).
- **Lines 32-42 (Editor directories and files)**:
  - **Basic Function**: Keep git clean from various OS and IDE temporary files.
  - **Detailed Explanation**: Excludes nested VS Code files (Line 33) but preserves extensions setup (Line 34), IDE projects (Line 35), system metadata (Line 36), Visual Studio user preference files (Line 37), Node.js Tooling files (Lines 38-39), solution manifests (Line 40), and temporary swap files (Line 41).
